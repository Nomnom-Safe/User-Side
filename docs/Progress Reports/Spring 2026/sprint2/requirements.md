# Sprint 2 Requirements

This document lists the updated requirements for each updated Sprint 2 goal.

> Note: Requirements and goals differ from those proposed in S1P.

- **Total goals:** 3
- **Total requirements:** 15

## Goals

1. Build an evidence-based user testing strategy for NomNom Safe's business side
2. Stabilize and polish the NomNom Safe user-side MVP for demo readiness
3. Prepare and deliver a complete final presentation and demonstration

## Requirements

1. Build an evidence-based user testing strategy for NomNom Safe's business side

   1.1 As a **researcher**, I want to collect AI-generated guidance on multiple user testing methods so that I can quickly compare options appropriate for web-app MVP validation.

   1.2 As a **researcher**, I want to consolidate testing method definitions, strengths, and weaknesses into a single reference so that sprint decisions are documented in one place.

   1.3 As a **researcher**, I want to evaluate testing strategies against NomNom Safe constraints (time, remote feasibility, and participant type) so that the chosen approach is realistic.

   1.4 As a **planner**, I want to identify a prioritized combination and sequence of testing methods for the business side so that work can proceed with a clear methodology.

   1.5 As a **student developer**, I want to document AI prompts/responses and resulting analysis artifacts so that rationale for strategy changes is transparent.

2. Stabilize and polish the NomNom Safe user-side MVP for demo readiness

   2.1 As a **developer**, I want to resolve Firebase/MCP access and verify Firestore schema alignment with models so that app data loads reliably.

   2.2 As a **developer**, I want to fix critical bugs and null-safety issues identified during demo prep so that common flows do not fail in front of users.

   2.3 As a **developer**, I want to address edge cases and improve error handling so that the app behaves predictably across failure states.

   2.4 As a **developer**, I want to standardize UI consistency (navigation patterns, spacing, headings, and loading indicators) so that the user experience feels cohesive.

   2.5 As a **developer**, I want to standardize user feedback patterns (errors, success states, empty states, and loading interactions) so that users receive clear, consistent guidance.

   2.6 As a **developer**, I want to complete code-quality cleanup and test updates tied to sprint fixes so that the codebase remains maintainable and verified.

   2.7 As a **developer**, I want to keep the user-side codebase stable while preparing presentation assets so that late sprint work does not introduce regressions.

3. Prepare and deliver a complete final presentation and demonstration

   3.1 As a **presenter**, I want to produce a user-side demonstration video so that core app behavior can be shown clearly during presentation.

   3.2 As a **presenter**, I want to build a final slide deck with project context, importance, and demo media so that NomNom Safe is communicated as a complete product.

   3.3 As a **capstone team member**, I want to deliver the final Celebration presentation for NomNom Safe so that sprint outcomes are formally presented.
